### Порядок действий в работе приложения
1. Frontend: Пользователь нажимает на кнопку и устнанавливаеться новое соединение
2. Bakckend: Сервер создает в списке новое соединение в списке соединений
3. Frontend: Посылается запрос для каждого типа данных на сервер в формате JSON: <br> 
    ```
    {
        "type": "upload/download/ping",
        "action:": "getData"
    }
    ```
4. Frontend: запускает имитацию анимации анализа трафика и ждет ответ от сервера
5. Backend: принимает запрос от Frontend и отправляет данные в формате JSON: <br>
    ```
    {
        "type": "upload/download/ping",
        "value": Данные в офрмате Int,
        "active": true/false (false врзвращаеть при отправке ping — завершение процесса передачи)
    }
    ```
6. Frontend: когда ```"active": false```, соединение разрываеться
7. Backend: когда разрываеться соедиение, проверить какие соединения еще активны и удалить все разорванные соединения из списка соединений
